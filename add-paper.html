How to add a paper

Catalog Venues

The site reads a single file, papers.bib, at the repository root. Add or edit BibTeX entries there, commit, and push; GitHub Pages (or any static host) will serve the updated bibliography on the next deploy.

1. Pick an entry type

Use @inproceedings for conference papers (typical) or @article for journal articles. Other types such as @misc (preprints) work; the parser records author, title, year, venue, link fields, and optional labels.

2. Fields the catalog uses

  • Authors — author = {Last1, First1 and Last2, First2} (BibTeX and separated).
  • Title — title = {...}; braces are stripped for display.
  • Year — year or leading four digits of date.
  • Venue — For conferences, booktitle is used first, then series, then journal. For articles, journal is preferred. Long publisher strings are mapped to short labels (for example ISCA, ASPLOS) via js/venue-aliases.js; add a pattern there if your venue does not match yet.
  • Link — url if present; otherwise doi becomes a DOI link; otherwise eprint can point to arXiv.
  • Tags — optional labels = {tag1, tag2} (commas or semicolons). They power the tag filters on the main page.

3. Example (conference)

@inproceedings{mykey2025,
  author    = {Doe, Jane and Smith, Alex},
  title     = {Learning Better Systems},
  booktitle = {Proceedings of the 52nd Annual International Symposium on Computer Architecture},
  year      = {2025},
  url       = {https://doi.org/10.1145/example},
  labels    = {systems, learning},
}

4. Common pitfalls

  • Percent signs — In BibTeX, % starts a comment. Inside abstracts or text, write \%.
  • Balanced braces — Every { in a field must close; a missing } can swallow the rest of the file.
  • Parse warning — If the catalog shows a count mismatch, check for stray @ inside fields, bare %, or a missing comma after the cite key line.

5. Optional staging file

If you keep a local future_papers.bib (or similar), copy finished entries into papers.bib when you want them on the site — only papers.bib is loaded.
